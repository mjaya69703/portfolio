## The Problem

I have multiple Laravel projects — NexaCampus (SIAKAD), Neco Siakad, Arpotret, and Laraflare — and I need them all accessible via different subdomains. Running each on a separate VPS would be expensive and wasteful. Instead, I consolidate everything on a single Mini PC using Proxmox VE with LXC containers.

## Architecture Overview

```
Mini PC (Windows 11 Pro — Intel i5-1250P, 24GB RAM)
│
└── VMware Workstation
    └── Proxmox VE
        │
        ├── LXC: Proxy
        │     └── Nginx + Cloudflare Tunnel
        │
        ├── LXC: Web Applications
        │     ├── NexaCampus (nexacampus.idev-fun.org)
        │     ├── Neco Siakad (siakad-pt.idev-fun.org)
        │     ├── NecoSiakad (neco-siakad.idev-fun.org)
        │     ├── Arpotret (arpotret.idev-fun.org)
        │     └── Laraflare (laraflare.idev-fun.org)
        │
        ├── LXC: Docker
        │     ├── Grafana + Prometheus
        │     ├── N8N, Vaultwarden, Affine
        │     └── Supporting Services
        │
        └── LXC: Database
              └── MariaDB
```

## Why Proxmox LXC Instead of Docker Compose?

LXC containers give me OS-level isolation — each container has its own filesystem, networking, and process space. For Laravel apps that need different PHP versions, extensions, or system dependencies, this separation is cleaner than managing multiple Docker services. LXC also has lower overhead than full VMs, so I can run more containers on limited hardware.

## Setting Up the Web Application LXC

Each Laravel app runs inside a single LXC container with Nginx as the web server and PHP-FPM for processing. The key configuration:

### 1. Nginx Server Blocks

Each app gets its own server block:

```nginx
server {
    listen 80;
    server_name nexacampus.idev-fun.org;
    root /var/www/nexacampus/public;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        fastcgi_pass unix:/run/php/php8.2-fpm.sock;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        include fastcgi_params;
    }
}
```

### 2. Database Separation

All apps connect to a dedicated Database LXC running MariaDB. Each app has its own database and user with minimal permissions:

```sql
CREATE DATABASE nexacampus CHARACTER SET utf8mb4;
GRANT ALL PRIVILEGES ON nexacampus.* TO 'nexacampus'@'10.0.0.%';
```

### 3. Cloudflare Tunnel for External Access

Instead of port forwarding, I use Cloudflare Tunnel on the Proxy LXC to securely expose all services:

```yaml
ingress:
  - hostname: nexacampus.idev-fun.org
    service: http://web-apps:80
  - hostname: arpotret.idev-fun.org
    service: http://web-apps:80
```

## Key Lessons Learned

1. **Resource limits matter** — set `memory` and `swap` limits on each LXC to prevent one app from starving others
2. **Backup strategy** — use `vzdump` for scheduled LXC snapshots plus `mysqldump` for database backups
3. **SSL is handled by Cloudflare** — no need to manage Let's Encrypt certificates internally since the tunnel handles TLS termination
4. **Deployment** — a simple `git pull` + `php artisan migrate` + `php artisan optimize:clear` on the target LXC, triggered via GitHub Actions webhook

## Monitoring

The Docker LXC runs Grafana and Prometheus, which scrape metrics from:
- **node_exporter** on each LXC (CPU, memory, disk)
- **nginx-prometheus-exporter** for request metrics
- **mysqld_exporter** for database health

Alerts are configured for high memory usage, disk space warnings, and service downtime.
