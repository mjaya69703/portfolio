# AscendKyo.dev - Website Specification

## Project Overview

Website personal milik Muhamad Jaya Kusuma (AscendKyo).

Tujuan website bukan sekadar portfolio, melainkan menjadi digital headquarters yang merepresentasikan perjalanan, proyek, homelab, eksperimen, dan aktivitas teknologi sehari-hari.

Website harus terasa seperti milik seseorang yang bekerja di bidang Infrastructure, IT Operations, Cloud, DevOps, dan Automation.

Hindari kesan:

* Freelancer pembuat website
* Agency digital
* Landing page template
* Portfolio mahasiswa

Target impression:

"Engineer yang suka membangun dan mengoperasikan sistem."

# User Personality Reflection

Website should reflect someone who:

* Started from technical support
* Grew into infrastructure operations
* Enjoys self-hosting
* Loves experimenting with technology
* Is building towards DevOps and Cloud Engineering

The website should feel authentic and operational, not like a marketing page.

---

# Final Rule

If a design decision is unclear:

Choose simplicity.

If a feature feels unnecessary:

Remove it.

The website should feel like a reliable Linux server:

Clean.
Efficient.
Purposeful.
